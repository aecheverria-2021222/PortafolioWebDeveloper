Contexto

El Sistema Bancario Multi-servicio es una plataforma que integra las operaciones típicas de una entidad bancaria —gestión de cuentas, usuarios y transacciones— en una arquitectura desacoplada, donde cada dominio se resuelve de forma independiente.

El proyecto se desarrolló en equipo, aplicando metodología SCRUM, con el objetivo de simular un producto real y no un simple ejercicio académico.

El problema

Las operaciones bancarias exigen seguridad, consistencia de los datos y capacidad de crecer sin que el sistema se vuelva imposible de mantener. El reto era diseñar una solución en la que cada servicio pudiera evolucionar por separado y el equipo trabajar en paralelo sin pisarse.

Mi rol

Participé en dos frentes: la parte técnica y la coordinación del equipo.

  • Definí la arquitectura de servicios y la separación por dominios.
  • Desarrollé funcionalidad tanto de backend como de frontend.
  • Como Scrum Master, facilité las ceremonias, gestioné el backlog y mantuve al equipo enfocado en entregas iterativas.

La solución

Estructuramos la plataforma en servicios independientes por dominio, con una base de datos relacional PostgreSQL para garantizar la integridad de las operaciones. Todo el entorno se empaquetó con Docker para que cualquier miembro del equipo pudiera levantar el sistema de forma idéntica.

  • Backend en Java y Node.js según el servicio.
  • Persistencia en PostgreSQL con un modelo de datos cuidado.
  • Contenedores Docker para un entorno reproducible.
  • Control de versiones con Git y flujo de trabajo en equipo.

Resultados y aprendizajes

El equipo entregó una plataforma funcional que cubría el flujo bancario principal. A nivel personal, fue el proyecto donde más aprendí sobre arquitectura desacoplada y, sobre todo, sobre liderazgo: coordinar a un equipo y traducir requisitos en tareas accionables.
